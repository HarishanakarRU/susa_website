This is the admin panel for the Statistics Undergraduate Student Association at the University of California, Berkeley.

There isn't much to say about creating new entries in our database, but there are several conventions that need to be followed when entering data for resources.

Resources

  • The type field denotes the type that you want the new entry to be displayed at.
    • 1 : goes under courses
    • 2 : goes under useful links
    • 3 : goes under other links
  • The position field denote the order that you want the new entry to be displayed under (within a group—explained next).
  • The category field denotes what category that you want the new entry to be displayed under—this is practically the same as group_position (but this doesn't give position). I created the two fields for coding/storage flexibility.
  • The label field denotes what you want the link text to display on the page
  • The link field denotes what you want the link to be.
    • If you have a link to an outside website, make sure to give the entire thing (http.......).
    • If you're linking to a resource that you upload, you only have to give the relative file path within /static/resources/.
  • The group_position field denotes what order that you want group that the entry belongs to to be in within its type.
